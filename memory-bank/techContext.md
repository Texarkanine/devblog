# Tech Context

Jekyll static site (Ruby, Bundler). Local dev: `bundle install` then `bundle exec jekyll serve`. Content is Markdown (Kramdown); HTML is compressed in production via jekyll-compress-html. Remote theme: riggraz/no-style-please; theme options in `_config.yaml` under `theme_config`.

## Build Tools

- **Jekyll** – `_config.yaml`, `Gemfile`; plugins listed under `plugins:`.
- **Kramdown** – Markdown engine; heading IDs are generated by default (anchor_ids).

## Testing Process

No automated test suite referenced in README; validation is manual (local serve, visual check). Linting/formatting not specified for this repo.

## Design System

Theme-driven (no-style-please). Local overrides via `_includes/`, `_layouts/`, and SASS if present. No separate design tokens or Storybook; keep CSS minimal when adding features.
